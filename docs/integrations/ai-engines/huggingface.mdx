Hugging Face facilitates building, training, and deploying ML models. Now you can create Hugging Face models within MindsDB.

<Note>
Please note that if you use local installation of MindsDB, instead of MindsDB Cloud, you should install `transformers==4.21.0` to be able to use the Hugging Face models.
</Note>

## How to Bring the Hugging Face Model to MindsDB

We use the [`CREATE MODEL`](/sql/create/model) statement to bring the Hugging Face models to MindsDB.

<Tabs>
<Tab title="Downloading Models from Hugging Face Repository">
One way is to download Hugging Face models from their repository. To do so, you need to specify the `model_name` parameter.

```sql
CREATE MODEL mindsdb.spam_classifier
PREDICT PRED
USING
  engine = 'huggingface',
  task = 'text-classification',
  model_name = 'mrm8488/bert-tiny-finetuned-sms-spam-detection',
  input_column = 'text_spammy',
  labels = ['ham', 'spam'];
```
</Tab>

<Tab title="Using Hugging Face Inference API">
Another way is to specify a task to be performed by the model and the Hugging Face Inference API does the rest.

```sql
CREATE MODEL mindsdb.sentiment_classifier
PREDICT sentiment
USING
  engine = 'hf_inference_api',
  task = 'text-classification',
  column = 'text';
```

Please note that in order to use the Hugging Face Inference API, you need to create the `hf_inference_api` ML engine using your Hugging Face token:

```sql
CREATE ML_ENGINE hf_inference_api
FROM huggingface_api
USING api_key = 'hf_xxx';
```

You can generate Hugging Face tokens in the `Setings -> Access Tokens` tab of your Hugging Face account.
</Tab>
</Tabs>

Let's go through some sample models.

<Tip>
Please note that the examples presented here use SQL. To see how to create Hugging Face models in Mongo database using MQL, check out [this example on sentiment classification](/nlp/nlp-mindsdb-hf#example-using-mql).
</Tip>

### Model 1: Spam Classifier

Here is an example of a binary classification. The model determines whether a text string is a spam or not.

<Tabs>
<Tab title="Downloading Models from Hugging Face Repository">
```sql
CREATE MODEL mindsdb.spam_classifier
PREDICT PRED
USING
  engine = 'huggingface',
  task = 'text-classification',
  model_name = 'mrm8488/bert-tiny-finetuned-sms-spam-detection',
  input_column = 'text_spammy',
  labels = ['ham', 'spam'];
```

Before querying for predictions, we should verify the status of the `spam_classifier` model.

```sql
DESCRIBE spam_classifier;
```

On execution, we get:

```sql
+---------------+-------+--------+--------+-------+-------------+---------------+------+-----------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|NAME           |PROJECT|STATUS  |ACCURACY|PREDICT|UPDATE_STATUS|MINDSDB_VERSION|ERROR |SELECT_DATA_QUERY|TRAINING_OPTIONS                                                                                                                                                                                               |
+---------------+-------+--------+--------+-------+-------------+---------------+------+-----------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|spam_classifier|mindsdb|complete|[NULL]  |PRED   |up_to_date   |22.10.2.1      |[NULL]|[NULL]           |{'target': 'PRED', 'using': {'engine': 'huggingface', 'task': 'text-classification', 'model_name': 'mrm8488/bert-tiny-finetuned-sms-spam-detection', 'input_column': 'text_spammy', 'labels': ['ham', 'spam']}}|
```

Once the status is `complete`, we can query for predictions.


</Tab>

<Tab title="Using Hugging Face Inference API">
```sql
CREATE MODEL mindsdb.spam_classifier
PREDICT PRED
USING
  engine = 'hf_inference_api',
  task = 'text-classification',
  column = 'text_spammy';
```

<Note>
Please note that the `text-classification` task uses two labels: `POSITIVE` and `NEGATIVE`.
</Note>
</Tab>
</Tabs>

### Model 3: Zero-Shot Classifier

Here is an example of a zero-shot classification. The model determines to which of the defined categories a text string belongs.

<Tabs>
<Tab title="Downloading Models from Hugging Face Repository">
```sql
CREATE MODEL mindsdb.zero_shot_tcd
PREDICT topic
USING
  engine = 'huggingface',
  task = 'zero-shot-classification',
  model_name = 'facebook/bart-large-mnli',
  input_column = 'text_short',
  candidate_labels = ['travel', 'cooking', 'dancing'];
```

Before querying for predictions, we should verify the status of the `zero_shot_tcd` model.

```sql
DESCRIBE zero_shot_tcd;
```

## HuggingFace + MindsDB Models Library

### Text Classification

#### Spam

Let's create a model.

```sql
CREATE MODEL mindsdb.hf_spam
PREDICT PRED
USING
 engine = 'huggingface',
 task = 'text-classification',
 model_name = 'mariagrandury/roberta-base-finetuned-sms-spam-detection',
 input_column = 'text',
 labels = ['spam', 'ham'];
```

And check its status.

```sql
DESCRIBE hf_spam;
```

Once the status is `complete`, we can query for predictions.

```sql
SELECT *
FROM mindsdb.hf_spam
WHERE text = 'I like you. I love you.';
```

On execution, we get:

```sql

+----+--------------------------------------------------------+-----------------------+
|spam|{"ham":0.00020051795581821352,"spam":0.9997995495796204}|I like you. I love you.|

```
